W PHP deklarujemy je słowem kluczowym class, po którym podajemy jej unikatową nazwę. Zasady jej tworzenia są podobne, jak w przypadku nazw zmiennych, tj. nie mogą one zaczynać się od cyfry. Następnie w nawiasach klamrowych umieszczamy informacje o dozwolonych polach oraz metodach, jakie klasa będzie posiadać:
                class NazwaKlasy
                {
                    public $atrybutl;
                    public $atrybut2;
                    function metodal (){
                        instrukcja1;
                        instrukcja2;
                        ...
                        instrukcjaN;
                    }
                    function metoda2( $paraml, $param2){
                        instrukcja1;
                        instrukcja2;
                        ...
                        instrukcjaN;
                    }
                }
Na podstawie zadeklarowanej k1asy można stworzyć obiekt czyli egzemplarz klasy. Obiekt tworzy się, stosując słowo k1uczowe new: 
                $obiekt = new NazwaKlasy();
Dla każdej klasy możemy zdefiniować specjalną metodę, zwaną konstruktorem. Konstruktor jest wywoływany przy tworzeniu obiektu, może więc wykonywać przydatną inicjalizację zadań, takich jak nadanie atrybutom początkowych wartości.
Konstruktor jest deklarowany w taki sam sposób jak inne operacje, ale posiada specjalną nazwę _construct(). Jego zadaniem jest automatyczne wywołanie podczas tworzenia obiektu. Deklaracja konstruktora może wyglądać następująco: 
                class NazwaKlasy
                {
                    public $atrybutl;
                    public $atrybut2;
                    function __construct($nowaWartosc1, $nowaWartosc2){
                        $atrybutl = $nowaWartosc1;
                        $atrybut2 = $nowaWartosc2;
                    }
                    function metodal (){
                        instrukcja1;
                        instrukcja2;
                        ...
                        instrukcjaN;
                    }
                    function metoda2( $paraml, $param2){
                        instrukcja1;
                        instrukcja2;
                        ...
                        instrukcjaN;
                    }
                }
W przeciwieństwie do funkcji i programowania strukturalnego, klasy posiadają precyzyjne mechanizmy kontroli dostępu do swojego wnętrza. Proces ukrywania części funkcjonalności przed programistą nosi nazwę hermetyzacji i pomaga zwiększyć niezawodność oprogramowania. Tworząc klasę, będziemy zawsze starali się określić tzw. publiczny interfejs, z którego może korzystać programista, odwołując się do tworzonych obiektów, jednocześnie ukrywając wszystkie wewnętrzne aspekty działania klasy. Interpreter będzie pilnował, aby użytkownicy nie wywołali żadnej "wewnętrznej" metody, co mogłoby doprowadzić do błędów w działaniu lub użycia jej niezgodnie z przeznaczeniem.
Modyfikatorami dostępu są:
  • public - atrybuty lub metody opisane tym modyfikatorem są widoczne na zewnątrz obiektu i mogą być modyfikowane w dowolnym miejscu kodu, zarówno wewnątrz klasy, wewnątrz klas potomnych oraz w obiekcie utworzonym na podstawie klasy
  • protected - atrybuty lub metody opisane tym modyfikatorem są widoczne tylko wewnątrz klasy i klas potomnych. Mogą być modyfikowane jedynie wewnątrz klasy, wewnątrz klas potomnych. Nie są widoczne w obiekcie utworzonym na podstawie klasy.
  • private - atrybuty lub metody opisane tym modyfikatorem są widoczne tylko wewnątrz klasy, w której zostały zdefiniowane i tylko w niej mogą być. modyfikowane. Nie są widoczne ani wewnątrz klas potomnych ani w obiekcie utworzonym na podstawie klasy.
